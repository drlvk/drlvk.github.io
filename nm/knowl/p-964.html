The line values = [f(T(:,1)) f(T(:,2)) f(T(:,3))] resists vectorization unless we rewrite f (which we probably should not do because in practice, the objective function is user-provided code). So it becomes a loop:

values = zeros(1, n+1);
for j = 1:n+1
    values(j) = f(T(:, j));
end

The remaining adjustments are small: the midpoint of opposite side is now M = (sum(T, 2) - T(:, ind))/n but the formulas for reflection \(R\text{,}\) expansion \(E\text{,}\) and contraction remain the same.
