Octave-specific remark. The current version of Octave has a similar function glpk instead of linprog. The main difference is that glpk(c, A, b) solves (35.1.3) instead of (35.1.1). However, the fourth argument of glpk can be used to set lower bounds to \(-\infty\) instead of \(0\text{,}\) the fifth argument (for upper bounds) can be left empty [], and the sixth argument is a string indicating the form of constraints, with letters 'U' for \(\le\text{.}\) So, ExampleÂ 35.1.1 with Octave would be

glpk([-1; -1], [1 2; 3 1], [4; 5], [-Inf; -Inf], [], 'UU')

See help glpk for more.
